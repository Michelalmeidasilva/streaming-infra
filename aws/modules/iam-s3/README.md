# AWS IAM S3 Access Module

This module provisions a dedicated IAM User tailored for the VOD platform's microservices. It automatically creates Access Keys and attaches a highly restrictive IAM Policy that allows operations *only* on the designated S3 bucket.

## Principle of Least Privilege
The policy generated by this module ensures that the backend and transcode workers can interact with the objects in the specified bucket for tasks such as Multipart Uploads, generating Presigned URLs, and retrieving files, with **no access** to other buckets or AWS services.

## Quick Start

### Usage with Terraform

Add this block to your main infrastructure setup, linking it with the S3 module:

```hcl
module "iam_s3" {
  source     = "./modules/iam-s3"
  user_name  = "vod-storage-svc"
  bucket_arn = module.storage_s3.bucket_arn
}
```

Deploy the infrastructure:

```bash
terraform init
terraform apply
```

To view the generated secret key (which is marked as sensitive) to add to your `.env` file, you can run:
```bash
terraform output -raw iam_secret_access_key
```

## Alternative: Manual Setup via AWS Console

If you prefer to create this user manually through the AWS Console, follow these steps:

### 1. Create the IAM Policy
1. Go to the [IAM Console](https://console.aws.amazon.com/iam/home) -> **Policies**.
2. Click **Create policy** and select the **JSON** tab.
3. Paste the following JSON (replace `YOUR_BUCKET_NAME` with your actual bucket name):
```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowS3Actions",
            "Effect": "Allow",
            "Action": [
                "s3:PutObject",
                "s3:GetObject",
                "s3:DeleteObject",
                "s3:ListBucket",
                "s3:AbortMultipartUpload",
                "s3:ListMultipartUploadParts",
                "s3:ListBucketMultipartUploads"
            ],
            "Resource": [
                "arn:aws:s3:::YOUR_BUCKET_NAME",
                "arn:aws:s3:::YOUR_BUCKET_NAME/*"
            ]
        }
    ]
}
```
4. Name the policy `vod-storage-policy` and save it.

### 2. Create the IAM User
1. Go to **Users** in the IAM Console and click **Add users**.
2. **User name**: `vod-storage-svc`. Click Next.
3. **Permissions options**: Select **Attach policies directly**.
4. Search for the `vod-storage-policy` you just created, check the box, and click Next -> **Create user**.

### 3. Generate Access Keys
1. Click on your newly created user `vod-storage-svc`.
2. Go to the **Security credentials** tab.
3. Under **Access keys**, click **Create access key**.
4. Select **Application running outside AWS** (since it's running in your local Docker/Vercel) and click Next -> **Create access key**.
5. **CRITICAL**: Copy the `Access key ID` and `Secret access key` immediately. You won't be able to see the secret key again. Paste them into your `.env` files.
