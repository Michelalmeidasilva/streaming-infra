# Tarefa 8.1: Shift-left antes do PR

## 📌 Épico
**Developer Experience**

## 🎯 Objetivo
Dar feedback de erros ao dev na própria máquina (IDE/pré-commit).

## 🧠 Contexto e Embasamento (RESEARCH)
A visão moderna de DevOps aplica o shift-left para o limite. Ao rodar verificações localmente, o desenvolvedor conserta problemas antes do push, poupando tempo.

## 📋 Requisitos
- Configurar pre-commit hooks
- Rodar lint local
- Rodar scan de segurança local
