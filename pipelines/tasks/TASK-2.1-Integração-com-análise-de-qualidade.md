# Tarefa 2.1: Integração com análise de qualidade

## 📌 Épico
**Code Quality & Code Smells**

## 🎯 Objetivo
Identificar e reportar code smells, complexidade e cobertura de código.

## 🧠 Contexto e Embasamento (RESEARCH)
Code smells não são necessariamente bugs, mas aumentam o risco de problemas futuros e dificultam a manutenção. O SonarQube medirá segurança, manutenibilidade, confiabilidade, cobertura, e complexidades ciclomática e cognitiva (conforme RESEARCH.md).

## 📋 Requisitos
- Integrar SonarQube ou SonarCloud
- Configurar análise no PR
- Configurar métricas: Code smells, Complexidade, Coverage
