# Tarefa 1.1: Configurar pipeline inicial

## 📌 Épico
**Pipeline Base (CI/CD)**

## 🎯 Objetivo
Ter pipeline rodando em PR e merge

## 🧠 Contexto e Embasamento (RESEARCH)
Uma pipeline moderna é o alicerce do DevSecOps. Ela não deve apenas buildar e testar, mas preparar o terreno para encontrar, bloquear e priorizar problemas (conforme RESEARCH.md, integrando Code Quality e Security desde o PR).

## 📋 Requisitos
- Criar pipeline no GitHub Actions
- Configurar triggers: Pull Request e Merge na main
- Adicionar etapa de build
- Adicionar execução de testes unitários
- Configurar cache de dependências
